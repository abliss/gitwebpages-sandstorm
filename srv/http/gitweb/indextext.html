To access this repository from a git client, first generate a webkey by clicking the icon on the Sandstorm top bar. The webkey will be of the form <endpointUrl>#<token>. The repository will then be available at <endpointUrl>/repo.git, protected by <token> as a password. You can use any username. The token not only provides authorization but also identifies this specific GitWeb instance.

For example, if endpointUrl is https://api.sandstorm.example.com and you want to name your repo "my_repo", you could do this, entering the token when prompted for a password:

$ git clone https://my_repo@api.sandstorm.example.com/repo.git my_repo

Note that we've included "my_repo" here as the username. This is so that if you use a git credential helper, it will be able to distinguish this repository from other repositories also hosted at https://api.sandstorm.example.com.
